# Glossary

- Candidate: a non-collapsed noisy-rate tower selected from the parent readout.
- Full available tower: all tiers produced by the current noisy-rate schema.
- Concrete step: a lifted abstract tower action that realizes a base transition.
- Learner update: an update summary emitted by the tower learner.
- Training health: whether execution, lift, concrete steps, tier use, and learner updates are coherent.
