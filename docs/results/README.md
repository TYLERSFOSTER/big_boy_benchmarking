# Results

Human-facing result summaries live here.

These pages summarize actual runs after artifacts exist. Machine-readable
artifacts remain the source of truth for pass/fail status, timings, events, and
diagnostics.

Result docs must not invent pass/fail status or scientific claims that are not
supported by recorded artifacts.

Most run interpretation should first live in the relevant repo-side evaluation
readout under `docs/evaluations/`. Use `docs/results/` only when promoting a
durable result summary beyond an evaluation-local readout.
