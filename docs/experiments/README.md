# Experiments

Human-facing experiment notes live here.

These pages describe experiment matrices and harness checks in prose. The
machine-readable artifact manifests and event tables remain the source of truth
for actual execution.

Experiment docs should name the environment ids, mode ids, seed policy, budget,
artifact root, and claim boundary.

Experiments are planning surfaces. When an experiment becomes a reusable
benchmark comparison, promote its design into an evaluation under
`docs/evaluations/` and satisfy the evaluation-construction protocol.
