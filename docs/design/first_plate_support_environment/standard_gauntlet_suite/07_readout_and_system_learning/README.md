# Readout And System Learning Design

## Purpose

This folder is for the human-readable output layer and system-learning archive
for the PlateSupport gauntlet.

## Starting Point

Counterpoint taught that machine tables are not enough: every evaluation needs
human-readable reports, badges, explicit claim boundaries, and durable design
memory for confusions or upstream/BBB integration lessons discovered during
runs.

## Design Question

What readout format lets future humans understand the entire gauntlet without
losing stage-level detail or confusing diagnostics with claims?
