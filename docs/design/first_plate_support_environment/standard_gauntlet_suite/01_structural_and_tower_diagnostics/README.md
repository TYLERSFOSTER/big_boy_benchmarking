# Structural And Tower Diagnostics Design

## Purpose

This folder is for the first child stage of the PlateSupport gauntlet: promote
environment-readiness facts into an evaluation-stage diagnostic surface where
later stages can consume stable structural and tower-shape evidence.

## Starting Point

The environment-readiness run already records candidate/valid state counts,
reachability, invalid moves, valid self-transitions, shortest path, default
tower depth, flat tower depth, and training-surface availability.

## Design Question

Which of those readiness artifacts should become formal suite-stage inputs, and
which additional diagnostics are needed before designing schema sweeps?
