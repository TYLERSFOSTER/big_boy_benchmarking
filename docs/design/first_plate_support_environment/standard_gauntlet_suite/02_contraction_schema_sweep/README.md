# Contraction Schema Sweep Design

## Purpose

This folder is for the PlateSupport analogue of the counterpoint contraction
diagnostics: vary schema or contraction strength to understand when towers are
flat, useful, noisy, degenerate, or structurally pathological.

## Starting Point

Counterpoint taught that source-local fraction semantics, noisy-rate semantics,
and first-projection collapse must be distinguished carefully. PlateSupport may
need different schema families because its geometry and action semantics are
not counterpoint-shaped.

## Design Question

What PlateSupport-specific schema sweep should identify candidate towers without
overfitting to counterpoint's edge-selection mechanics?
