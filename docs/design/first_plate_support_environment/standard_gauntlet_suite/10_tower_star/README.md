# PlateSupport Tower-Star Guarded Lift Comparison

## Purpose

This folder holds the design thread for the `tower_star` follow-up to the
PlateSupport direct-star cul-de-sac control diagnostic.

The direct-star diagnostic answered Abdul Malik's concern that the raw direct
PlateSupport baseline was being penalized by one-step invalid moves and
self-loop cul-de-sacs that the tower arm avoided through its executable lift
machinery. The next question, requested by the Project Owner in the
direct-star readout conversation, is whether the comparison should explicitly
"star" the tower side too.

This folder is for that design. It should not be treated as an implementation
approval until a Phase.Stage.Action implementation workplan exists and is
explicitly executed.

## Documents

- `01_001_plate_support_tower_star_guarded_lift_comparison_blueprint.md`:
  detailed blueprint for the first `tower_star` diagnostic evaluation.
- `01_002_plate_support_tower_star_guarded_lift_comparison_implementation_workplan.md`:
  Phase.Stage.Action implementation workplan derived from the blueprint.
