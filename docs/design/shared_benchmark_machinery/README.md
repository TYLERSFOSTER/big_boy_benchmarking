# Shared Benchmark Machinery Design

## Status

Design workspace.

## Scope

This folder is for designing the shared benchmark machinery that counterpoint
and later environment families should build on:

- artifact writers;
- mode registry;
- seed bundles;
- metric and event rows;
- timing helpers;
- runner skeletons;
- upstream integration;
- CLI.

## Boundary

This folder is for design artifacts only until the Project Owner approves a
Phase.Stage.Action implementation gameplan.

The paused counterpoint implementation should resume only after this shared
machinery exists and its prerequisite gate passes.
