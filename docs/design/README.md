# Design

This folder is open-lab engineering memory for `big_boy_benchmarking`.

It contains design discussions, blueprints, Phase.Stage.Action workplans,
implementation logs, system-learning archives, and release-readiness notes.

## Document Types

| Type | Purpose |
| --- | --- |
| Design discussion | Turn-by-turn or exploratory design conversation. Project Owner and Codex turns must be attributed truthfully. |
| Blueprint | Detailed design specification. Blueprints are not conversation transcripts and must not invent Project Owner turns. |
| Workplan | Phase.Stage.Action implementation plan derived from a blueprint. Once approved for execution, the workplan is the implementation authority. |
| Implementation log | Record of actual actions, commands, files changed, tests run, and stop conditions encountered. |
| System-learning archive | Durable notes about issues discovered through evaluation work and how they changed the system. |

## Public Open-Lab Note

These documents preserve live engineering reasoning and corrections. They are
public engineering memory, not polished papers. Preserve PO attribution, label
consultant inferences as inferences, and use `[XXX]` for public profanity
redaction when release hygiene requires it.
